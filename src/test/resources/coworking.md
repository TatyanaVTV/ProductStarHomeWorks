# Архитектура системы бронирования рабочих мест в коворкинге (CoworkingSystem)

## Краткое описание
Программа реализует систему управления бронированием рабочих мест в коворкинге. Она позволяет:
- добавлять и удалять рабочие места;
- регистрировать пользователей;
- бронировать и отменять бронирование рабочих мест;
- просматривать список доступных рабочих мест.

## Ключевые компоненты

### 1. Классы сущностей

#### User 
Представляет пользователя системы. Содержит:
- id (UUID) — уникальный идентификатор пользователя;
- name, surname — имя и фамилия пользователя;
- bookedWorkspaces (TreeSet<Workspace>) — множество забронированных пользователем рабочих мест (упорядочено).

Особенности:
- реализует интерфейс Comparable<User> для сортировки по фамилии, затем по имени;
- предоставляет неизменяемое представление списка забронированных мест через getBookedWorkspaces();
- содержит методы bookWorkspace() и cancelBooking() для управления бронированием.

#### Workspace
Представляет рабочее место. Содержит:
- number (int) — номер рабочего места;
- type (String) — тип рабочего места (например, «open space», «кабинет»);
- isAvailable (boolean) — флаг доступности рабочего места.

Особенности:
- реализует интерфейс Comparable<Workspace> для сортировки по номеру;
- содержит методы markAsBooked() и markAsAvailable() для изменения статуса доступности.

### 2. Классы исключений

#### WorkspaceException
Исключения, связанные с рабочими местами:
- alreadyExists — рабочее место уже существует в системе;
- unknownWorkspace — рабочее место не найдено в системе;
- alreadyFree — рабочее место уже свободно;
- isNotBookedByUser — рабочее место не забронировано указанным пользователем;
- alreadyBooked — рабочее место уже забронировано (с указанием пользователя или без).  
 
#### UserException
Исключения, связанные с пользователями:
- alreadyRegistered — пользователь уже зарегистрирован в системе;
- unknownUser — пользователь не найден в системе.

### 3. Основной класс системы
CoworkingSystem — центральный класс, управляющий всеми операциями. Содержит:
- workspaces (TreeSet<Workspace>) — упорядоченное множество всех рабочих мест в системе;
- bookingMap (TreeMap<User, TreeSet<Workspace>>) — карта бронирований: 
  - ключ (User) - пользователь
  - значение (TreeSet<Workspace>) - множество забронированных данным пользователем рабочих мест (упорядоченные структуры обеспечивают сортировку).

## Основные методы системы
- addWorkspace(Workspace workspace)
  - Добавляет новое рабочее место в систему
    - WorkspaceException.alreadyExists, если такое рабочее место уже есть в системе
- removeWorkspace(Workspace workspace)
  - Удаляет рабочее место из системы
    - WorkspaceException.unknownWorkspace, если указанное рабочее место не найдено в системе
- registerUser(User user)
  - Регистрирует нового пользователя
    - UserException.alreadyRegistered, если такой пользователь уже зарегистрирован в системе
- bookWorkspace(User user, Workspace workspace)
  - Бронирует рабочее место для пользователя
    - UserException.unknownUser, если такой пользователь не зарегистрирован в системе
    - WorkspaceException.unknownWorkspace, если указанное рабочее место не найдено в системе
    - WorkspaceException.alreadyBooked, если указанное рабочее место уже забронировано
- cancelBooking(User user, Workspace workspace)
  - Отменяет бронирование рабочего места
    - UserException.unknownUser, если такой пользователь не зарегистрирован в системе
    - WorkspaceException.unknownWorkspace, если указанное рабочее место не найдено в системе
    - WorkspaceException.isNotBookedByUser, если указанное рабочее место не забронировано за указанным пользователем
- printAvailableWorkingSpaces()	
  - Выводит список доступных рабочих мест
