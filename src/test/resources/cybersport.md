# Архитектура системы рейтингов игроков

## Краткое описание
Система предназначена для управления рейтингами игроков: добавления игроков, обновления их рейтингов и получения топ‑списка игроков. Реализована на Java с использованием стандартных коллекций и принципов ООП.

## Ключевые компоненты

### 1. Класс Player
Представляет игрока с уникальными идентификатором и именем, а также изменяемым рейтингом.

#### Поля:
- id (final) — уникальный идентификатор игрока;
- name (final) — имя игрока;
- rating — текущий рейтинг (изменяется со временем).

#### Ключевые методы:
- equals() — сравнивает игроков по id и name;
- hashCode() — генерирует хеш на основе id и name;
- compareTo() — сравнивает игроков по рейтингу (для сортировки);
- toString() — возвращает строковое представление игрока.

#### Особенности:
- Использует аннотации Lombok (@RequiredArgsConstructor, @Data) для сокращения шаблонного кода;
- реализует Comparable<Player> для возможности сортировки.

### 2. Исключение RankingSystemException
Пользовательское исключение для обработки ошибок в системе рейтингов.

#### Типы ошибок:
- игрок уже добавлен (playerAlreadyRanked);
- игрок не найден в рейтинговой системе (playerNotRanked).

#### Особенности:
- наследуется от RuntimeException;
- предоставляет статические фабричные методы для создания исключений с предопределёнными сообщениями.

### 3. Класс RankingSystem
Основной класс системы, управляющий рейтингами игроков.

#### Структура данных:
- playerRankings — TreeMap<Integer, TreeSet<Player>>:
  - ключ (Integer) — рейтинг игрока;
  - значение (TreeSet<Player>) — множество игроков с данным рейтингом;

TreeMap сортирует рейтинги по возрастанию;
TreeSet сортирует игроков внутри группы по рейтингу (через compareTo).

#### Ключевые методы:
- addPlayer(Player player) — добавляет нового игрока с рейтингом 0;
- updatePlayerRating(int playerId, int newRating) — обновляет рейтинг указанного игрока;
- getTopPlayers(int n) — возвращает топ‑N игроков по рейтингу;
- getPlayerRank(int playerId) — получает текущий рейтинг игрока.

## Логика работы:
- при добавлении игрок помещается в группу рейтинга 0;
- при обновлении рейтинга игрок перемещается между группами: удаляется из старой, добавляется в новую;
- рейтинг хранится как в Player.rating, так и в структуре playerRankings.
